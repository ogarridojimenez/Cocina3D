# spec/ — Cocina3D

> Proyecto gestionado con Spec-Driven Development (SDD).
> La especificación es la fuente de verdad. El código es derivado.

## Estructura

- `constitution/` — Misión, stack, roadmap (se define una vez)
- `features/` — Features por número correlativo

## Reglas

1. Primero se escribe la spec, luego el plan, luego las tareas, luego el código
2. Si algo cambia, se edita primero la especificación
3. Cada feature debe respetar la constitución
